# 06_candidate_matching.md

## 목표

추천 요청에 대해 촉탁의사 후보를 검색하고, 복수 추천 후보로 저장하는 기능을 구현한다.

---

## 핵심 원칙

시스템은 후보를 자동 확정하지 않는다.  
의사회 담당자가 후보를 검토하고 최종 추천 후보를 선택한다.

---

## 포함 기능

- 추천 요청 상세에서 후보 검색
- 촉탁의사 조건 필터
- 추천 가능 상태 우선 표시
- 교육 이수자 우선 표시
- 현재 담당 시설 수 표시
- 추천 후보 선택
- 추천 순위 지정
- 추천 사유 입력
- 추천 후보 저장
- 후보추천완료 상태 변경

---

## 후보 검색 기준

| 기준 | 설명 |
|---|---|
| 의사 상태 | AVAILABLE 우선 |
| 교육 상태 | COMPLETED 우선 |
| 전문과목 | 요청 전문과목 일치 |
| 활동 가능 지역 | 요양원 주소 또는 지역과 일치 |
| 현재 담당 시설 수 | 낮을수록 우선 |
| 최대 담당 시설 수 | 초과 시 경고 |
| 민원 이력 | 존재 시 표시 |
| 보류/중지 상태 | 기본 검색 제외 |

---

## 추천 점수

초기 MVP에서는 복잡한 알고리즘이 필요 없다.  
단순 점수 또는 정렬 기준으로 충분하다.

예시:

```text
기본 0점
교육 이수 완료 +30
상태 추천가능 +30
전문과목 일치 +20
가능 지역 일치 +10
담당 시설 수 3곳 이하 +10
보류/중지 상태 -100
```

점수는 참고용이며 최종 판단은 사람이 한다.

## 후보 목록 컬럼

| 컬럼 | 설명 |
|---|---|
| 선택 | 체크박스 |
| 추천 순위 | 숫자 |
| 이름 | doctor.name |
| 전문과목 | specialty |
| 소속 의료기관 | medicalInstitutionName |
| 가능 지역 | availableRegions |
| 교육 상태 | educationStatus |
| 현재 담당 시설 | currentFacilityCount |
| 상태 | status |
| 참고 점수 | matchingScore |
| 추천 사유 | 입력 |

---

## 추천 후보 저장 규칙

- 최소 1명 이상 선택
- 권장 2~3명 이상 선택
- 같은 요청에 같은 의사 중복 저장 금지
- 보류/중지 의사는 저장 시 경고 또는 차단
- 저장 후 요청 상태를 `CANDIDATES_SELECTED`로 변경 가능

---

## 구현 지시

1. 추천 요청 상세에 후보 검색 영역을 추가한다.
2. 후보 검색 API를 구현한다.
3. 추천 점수 계산 로직을 구현한다.
4. 후보 선택 및 순위 저장을 구현한다.
5. 저장된 후보 목록을 상세 화면에 표시한다.
6. 후보 저장 시 AuditLog를 남긴다.
7. 후보 저장 후 상태 변경을 연결한다.

---

## 예외 처리

- 후보 0명 선택
- 중복 후보 선택
- 존재하지 않는 의사 선택
- 중지 상태 의사 선택
- 이미 종결된 추천 요청 수정
- 권한 없는 후보 저장
