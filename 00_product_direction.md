# 00_product_direction.md

## 목표

개발 시작 전 서비스 방향, 범위, 금지 영역, 핵심 사용자, MVP 우선순위를 확정한다.

이 파트에서는 코드를 크게 작성하기보다, 프로젝트 내부에 제품 방향 문서를 반영하고 개발자가 헷갈리지 않게 기준을 세운다.

---

## 제품 정의

서비스명 임시명:

> 촉탁의 추천관리 시스템

서비스 정의:

> 지역의사협의회가 요양원의 촉탁의사 추천 요청부터 촉탁의 후보 검토, 복수 추천, 최종 지정, 계약 현황, 민원·재추천까지 관리하는 관리자용 행정관리 솔루션

---

## 핵심 방향

### 1. 관리자 중심 MVP

초기 버전은 의사회 사무국이 사용하는 내부 관리자 시스템이다.

요양원 로그인, 촉탁의사 로그인은 이번 단계에서 제외한다.

### 2. 행정관리 솔루션

본 시스템은 의료행위에 개입하지 않는다.

포함:

- 촉탁의사 후보 DB
- 요양원 DB
- 추천 요청 관리
- 추천 후보 관리
- 추천서 생성
- 지정 결과 관리
- 계약 기간 관리
- 민원·재추천 관리
- 대시보드

제외:

- 환자별 진료기록
- 처방
- EMR
- 공단 청구
- 의료 상담 AI
- 의료 판단 자동화

### 3. 검토 보조, 자동 확정 금지

시스템은 후보를 자동 정렬하거나 추천할 수 있으나, 최종 추천 확정은 의사회 담당자가 한다.

---

## 핵심 업무 흐름

```text
촉탁의사 등록
↓
교육 이수 확인
↓
요양원 등록
↓
추천 요청 접수
↓
후보 검색/검토
↓
복수 추천
↓
추천서 생성
↓
요양원 최종 지정
↓
계약 등록
↓
만료/민원/재추천 관리
```

## 구현 지시

1. 현재 프로젝트 루트에 `docs` 폴더가 없으면 생성한다.
2. `docs/product_direction.md` 파일을 만든다.
3. 본 문서의 핵심 내용을 정리해 저장한다.
4. README가 있다면 이 시스템의 목적을 한 단락으로 추가한다.
5. 기존 프로젝트가 있다면 기존 구조를 훼손하지 않는다.

## 용어 정의

| 용어 | 의미 |
|---|---|
| 촉탁의사 | 요양원과 계약 또는 지정되어 정기적으로 건강관리 업무를 수행하는 의사 |
| 요양원 | 촉탁의 추천을 요청하는 장기요양시설 |
| 지역의사협의회 | 촉탁의 추천·관리 행정 주체 |
| 추천 요청 | 요양원이 촉탁의 후보 추천을 요청하는 업무 |
| 복수 추천 | 의사회가 여러 명의 후보 의사를 추천하는 행위 |
| 지정 결과 | 요양원이 추천 후보 중 최종 선택한 의사 |
| 계약 | 지정된 촉탁의사와 요양원의 운영 기간 |
| 민원 | 요양원/의사 간 소통, 일정, 행정 문제 |
| 재추천 | 기존 추천 또는 지정이 어려울 때 다시 후보 추천하는 업무 |
