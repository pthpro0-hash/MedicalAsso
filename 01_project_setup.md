# 01_project_setup.md

## 목표

프로젝트 개발 환경과 기본 구조를 정리한다.  
기존 코드베이스가 있다면 그 구조를 존중하고, 없으면 관리자 웹앱 기준으로 새 프로젝트를 구성한다.

---

## 권장 구조

```text
/
├─ docs/
│  ├─ product_direction.md
│  └─ dev_notes.md
├─ src/
│  ├─ app/ 또는 pages/
│  ├─ components/
│  ├─ features/
│  ├─ lib/
│  ├─ server/
│  ├─ styles/
│  └─ types/
├─ prisma/ 또는 db/
├─ public/
├─ uploads/
└─ tests/
```

---

## 기술 스택 원칙

기존 프로젝트가 있으면 기존 기술을 우선한다.

새로 시작할 경우 권장:

- Next.js
- TypeScript
- Tailwind CSS
- Prisma
- PostgreSQL 또는 SQLite 개발환경
- Zod 또는 유사 검증 라이브러리
- Playwright 또는 기본 테스트 도구

---

## 공통 UI 원칙

관리자 시스템이므로 화려함보다 명확성이 중요하다.

- 좌측 사이드바
- 상단 페이지 제목
- 검색/필터 영역
- 테이블 목록
- 상세 페이지
- 상태 배지
- 주요 액션 버튼
- 변경 이력 영역

---

## 공통 메뉴

```text
대시보드
촉탁의사 관리
요양원 관리
추천 요청 관리
지정/계약 현황
민원·재추천

```

## 환경 변수 예시

```env
DATABASE_URL=
UPLOAD_DIR=./uploads
APP_NAME=촉탁의 추천관리 시스템
```

---

## 구현 지시

1. 저장소 구조를 확인한다.
2. 패키지 매니저를 확인한다.
3. 실행 스크립트를 확인한다.
4. 타입스크립트 사용 여부를 확인한다.
5. DB 설정 방식을 확인한다.
6. 없는 경우 기본 프로젝트 구조를 생성한다.
7. `docs/dev_notes.md`를 생성한다.
8. 기본 레이아웃과 라우팅 구조를 준비한다.
9. 공통 상태 배지, 테이블, 폼 컴포넌트의 위치를 정한다.
