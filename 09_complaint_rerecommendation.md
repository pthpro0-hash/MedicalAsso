# 09_complaint_rerecommendation.md

## 목표

촉탁의사 운영 중 발생하는 민원, 이슈, 재추천 요청을 기록하고 처리하는 기능을 구현한다.

---

## 포함 기능

- 민원 목록
- 민원 등록
- 민원 상세
- 민원 상태 변경
- 관련 요양원 연결
- 관련 촉탁의사 연결
- 관련 계약 연결
- 처리 메모
- 재추천 필요 표시
- 재추천 요청 생성 연결

---

## 민원 유형

```text
SCHEDULE = 방문 일정 문제
COMMUNICATION = 소통 문제
DOCUMENT = 서류 문제
SCOPE_MISUNDERSTANDING = 진료 범위 오해
GUARDIAN = 보호자 민원
CONTRACT_CHANGE = 계약 변경
OTHER = 기타
```

---

## 민원 상태

```text
RECEIVED = 접수
UNDER_REVIEW = 검토중
CHECKING = 확인중
ADJUSTED = 조정완료
RERECOMMENDATION_NEEDED = 재추천필요
CLOSED = 종결
```

---

## 민원 등록 필드

필수:

- 제목
- 민원 유형
- 내용
- 접수일
- 관련 요양원

선택:

- 관련 촉탁의사
- 관련 계약
- 접수자
- 처리 메모
- 첨부파일

---

## 재추천 연결

민원 상태가 `RERECOMMENDATION_NEEDED`가 되면 추천 요청을 새로 만들 수 있어야 한다.

생성되는 추천 요청:

- facilityId 자동 연결
- requestType = RERECOMMENDATION
- reason = 민원 내용 요약
- 기존 assignment 연결 가능
- 상태 = RECEIVED

---

## 주의사항

민원 관리는 의료 판단 기록이 아니다.  
진료 내용, 처방 내용, 환자 개인정보를 상세하게 입력하지 않도록 안내 문구를 표시한다.

권장 안내 문구:

```text
이 화면은 촉탁의 추천·계약·소통 관련 행정 민원 관리를 위한 화면입니다.
환자 진료기록, 처방 내용, 민감한 건강정보는 입력하지 마세요.
```

---

## 구현 지시

1. 민원 CRUD API를 구현한다.
2. 민원 목록 페이지를 만든다.
3. 민원 등록/상세/수정 화면을 만든다.
4. 요양원/의사/계약 검색 연결을 제공한다.
5. 상태 변경 기능을 구현한다.
6. 재추천 요청 생성 버튼을 구현한다.
7. 재추천 요청 생성 시 RecommendationRequest를 만든다.
8. 민원/재추천 이력을 AuditLog에 남긴다.
9. 환자 개인정보 입력 금지 안내를 UI에 표시한다.

---

## 예외 처리

- 관련 요양원 없는 민원 등록
- 종결 민원 수정
- 존재하지 않는 계약 연결
- 재추천 요청 중복 생성
- 의료정보로 보이는 필드 추가 금지
- 권한 없는 상태 변경
