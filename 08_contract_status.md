# 08_contract_status.md

## 목표

요양원이 최종 지정한 촉탁의사와 계약 기간을 등록하고, 계약 현황 및 만료 예정 건을 관리한다.

---

## 포함 기능

- 최종 지정 의사 선택
- 계약 시작일/종료일 등록
- 계약 상태 관리
- 계약서 파일 첨부
- 등록비/행정비 관리
- 만료 예정 계산
- 계약 목록
- 시설별/의사별 계약 이력

---

## 계약 상태

```text
ACTIVE = 진행중
EXPIRING_SOON = 만료예정
EXPIRED = 만료
TERMINATED = 해지
RENEWAL_PENDING = 재계약대기
```

---

## 지정 결과 등록 필드

필수:

- 추천 요청
- 요양원
- 최종 지정 의사
- 계약 시작일
- 계약 종료일

선택:

- 계약서 파일
- 등록비
- 등록비 납부 여부
- 메모

---

## 상태 변경 규칙

1. 추천 후보에 포함된 의사 중에서 지정하는 것을 기본으로 한다.
2. 추천 후보 외 의사를 지정하는 경우 경고를 표시한다.
3. 계약 시작일은 종료일보다 빨라야 한다.
4. 계약 등록 후 추천 요청 상태를 `CONTRACT_REGISTERED`로 변경할 수 있다.
5. 계약 종료 30일 이내이면 만료 예정으로 표시한다.
6. 종료일이 지났으면 만료로 표시한다.

---

## 계약 목록 컬럼

| 컬럼 | 설명 |
|---|---|
| 요양원 | facility.name |
| 촉탁의사 | doctor.name |
| 전문과목 | doctor.specialty |
| 계약 시작일 | contractStartDate |
| 계약 종료일 | contractEndDate |
| 상태 | status |
| 등록비 | registrationFee |
| 납부 여부 | registrationFeePaid |
| 남은 일수 | computed |

---

## 만료 예정 기준

기본 기준:

- 90일 이내
- 60일 이내
- 30일 이내
- 만료

대시보드에서는 30일 이내를 우선 노출한다.

---

## 구현 지시

1. 지정 결과 등록 API를 구현한다.
2. 추천 요청 상세에서 최종 지정 의사를 선택할 수 있게 한다.
3. 계약 시작일/종료일을 입력받는다.
4. Assignment 레코드를 생성한다.
5. 계약 목록 페이지를 만든다.
6. 만료 예정 계산 로직을 만든다.
7. 요양원 상세과 촉탁의사 상세에 계약 이력을 표시한다.
8. 등록비/납부 여부를 관리한다.
9. 지정/계약 등록 시 AuditLog를 남긴다.

---

## 예외 처리

- 추천 요청 없이 계약 등록
- 존재하지 않는 의사/요양원 선택
- 계약 종료일이 시작일보다 빠름
- 중복 활성 계약 경고
- 종결된 추천 요청 수정
- 권한 없는 계약 등록
