# 10_dashboard_report.md

## 목표

관리자 대시보드, 주요 통계, 업무 큐, 최종 통합 검수를 구현한다.

---

## 대시보드 목적

의사회 사무국이 오늘 처리해야 할 업무를 한눈에 확인하게 한다.

---

## 필수 지표

| 지표 | 설명 |
|---|---|
| 전체 촉탁의사 수 | Doctor 전체 |
| 추천 가능 촉탁의사 수 | status = AVAILABLE |
| 교육 이수 완료자 수 | educationStatus = COMPLETED |
| 전체 요양원 수 | Facility 전체 |
| 미배정 요양원 수 | 활성 계약 없는 시설 |
| 추천 요청 대기 | status = RECEIVED |
| 검토 중 요청 | status = UNDER_REVIEW |
| 후보 추천 완료 | status = CANDIDATES_SELECTED |
| 지정 완료 | status = ASSIGNED |
| 계약 만료 30일 이내 | contractEndDate 기준 |
| 민원 처리 대기 | Complaint closed 제외 |
| 재추천 필요 건 | status = RERECOMMENDATION_NEEDED |

---

## 업무 큐

대시보드에는 아래 목록을 표시한다.

### 1. 오늘 처리할 추천 요청

- 접수 상태
- 검토중 상태
- 후보 추천 후 발송 전 상태

### 2. 계약 만료 예정

- 30일 이내
- 60일 이내
- 90일 이내

### 3. 민원 처리 대기

- 접수
- 검토중
- 확인중
- 재추천필요

### 4. 확인 필요한 촉탁의사

- 교육 미확인
- 추천 가능 상태 아님
- 담당 시설 수 초과

---

## 대시보드 UI

- 상단 KPI 카드
- 업무 큐 테이블
- 상태별 추천 요청 요약
- 계약 만료 예정 목록
- 민원 처리 대기 목록
- 빠른 액션 버튼

빠른 액션:

```text
촉탁의사 등록
요양원 등록
추천 요청 등록
민원 등록
계약 현황 보기
```

---

## 리포트/내보내기

MVP 선택 기능:

- 촉탁의사 목록 CSV
- 요양원 목록 CSV
- 계약 현황 CSV
- 추천 요청 현황 CSV

CSV 기능이 부담되면 이번 파트에서 제외하고 TODO로 남겨도 된다.

## 최종 통합 검수 시나리오

아래 시나리오를 실제 UI에서 수행한다.

### 시나리오 1: 신규 추천 업무

1. 관리자 로그인
2. 촉탁의사 2명 등록
3. 요양원 1곳 등록
4. 추천 요청 등록
5. 후보 의사 검색
6. 후보 2명 선택
7. 추천서 PDF 생성
8. 발송 완료 처리
9. 최종 지정 의사 선택
10. 계약 기간 등록
11. 대시보드에서 지정/계약 상태 확인

### 시나리오 2: 계약 만료 관리

1. 계약 종료일이 30일 이내인 계약 생성
2. 계약 현황에서 만료 예정 표시 확인
3. 대시보드 만료 예정 목록 확인

### 시나리오 3: 민원과 재추천

1. 기존 계약에 민원 등록
2. 민원 상태를 재추천필요로 변경
3. 재추천 요청 생성
4. 추천 요청 목록에 재추천 건 표시 확인

### 시나리오 4: 권한 검수

1. READ_ONLY 계정으로 로그인
2. 데이터 수정 시도
3. 수정 차단 확인
