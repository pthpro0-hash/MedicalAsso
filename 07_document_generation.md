# 07_document_generation.md

## 목표

추천 요청에 저장된 후보를 기반으로 추천서/공문 PDF를 생성하고 문서 이력을 관리한다.

---

## 포함 기능

- 추천서 미리보기
- 추천서 PDF 생성
- 생성 문서 다운로드
- 생성 문서 이력 저장
- 추천 후보 목록 자동 반영
- 의사회 정보/담당자 정보 반영
- 추천 요청 상태 변경

---

## 생성 문서 1차 범위

### 촉탁의사 추천서

필수 포함 내용:

- 수신 요양원명
- 문서 제목
- 추천 요청일
- 추천일
- 추천 후보 목록
- 후보별 이름, 전문과목, 소속 의료기관, 연락처
- 지역의사협의회명
- 담당자
- 안내 문구

---

## 추천서 기본 문안

```text
수신: {{facilityName}}
제목: 촉탁의사 추천의 건

귀 시설의 촉탁의사 추천 요청에 따라 아래와 같이 촉탁의사 후보자를 추천합니다.

1. {{doctorName}} / {{specialty}} / {{medicalInstitutionName}} / {{phone}}
2. {{doctorName}} / {{specialty}} / {{medicalInstitutionName}} / {{phone}}
3. {{doctorName}} / {{specialty}} / {{medicalInstitutionName}} / {{phone}}

추천 후보 중 최종 지정 결과를 지역의사협의회로 회신하여 주시기 바랍니다.

추천일: {{today}}
지역의사협의회: {{associationName}}
담당자: {{managerName}}
```

---

## 문서 템플릿 관리

초기 MVP에서는 템플릿을 코드 또는 설정 파일로 관리해도 된다.  
향후 관리자 화면에서 수정 가능하도록 확장한다.

---

## PDF 생성 방식

권장:

- HTML 템플릿 생성
- PDF 렌더링
- 파일 저장
- Document 테이블에 메타데이터 저장

대안:

- 서버 PDF 라이브러리 사용
- 브라우저 print CSS 사용

---

## 문서 상태 연계

추천서 생성 후 사용자가 “발송 완료”를 누르면 추천 요청 상태를 아래로 변경한다.

```text
CANDIDATES_SELECTED → SENT_TO_FACILITY
```

초기 MVP에서는 실제 이메일 발송은 제외하고, 수동 발송 완료 체크만 구현한다.

---

## 구현 지시

1. 추천 요청 상세에 추천서 생성 버튼을 추가한다.
2. 추천서 미리보기 화면 또는 모달을 만든다.
3. 추천 후보 데이터를 템플릿에 바인딩한다.
4. PDF 생성 기능을 구현한다.
5. 생성된 파일을 저장하고 Document 레코드를 만든다.
6. 생성 문서 목록을 추천 요청 상세에 표시한다.
7. 발송 완료 처리 버튼을 만든다.
8. 문서 생성/발송 완료 이력을 AuditLog에 남긴다.

---

## 예외 처리

- 추천 후보가 없는 경우 PDF 생성 금지
- 요양원 정보가 없는 경우 생성 금지
- 파일 저장 실패
- PDF 생성 실패
- 종결/취소된 요청에서 문서 생성 시도
- 권한 없는 문서 생성
