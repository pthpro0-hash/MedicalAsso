# 05_recommendation_request.md

## 목표

요양원의 촉탁의 추천 요청을 등록하고 상태 기반으로 관리하는 기능을 구현한다.

---

## 추천 요청의 의미

추천 요청은 요양원이 지역의사회에 촉탁의 후보 추천을 요청한 업무 건이다.

초기 MVP에서는 요양원이 직접 입력하지 않고, 의사회 사무국 직원이 관리자 화면에서 대리 등록한다.

---

## 포함 기능

- 추천 요청 목록
- 추천 요청 등록
- 추천 요청 상세
- 상태 변경
- 요청 사유 기록
- 요청 시설 연결
- 희망 전문과목/요일 기록
- 첨부파일 연결
- 처리 메모
- 상태 변경 이력

---

## 추천 요청 상태

```text
RECEIVED = 접수
UNDER_REVIEW = 검토중
CANDIDATES_SELECTED = 후보추천완료
SENT_TO_FACILITY = 요양원선택대기
ASSIGNED = 지정완료
CONTRACT_REGISTERED = 계약등록완료
CLOSED = 종결
CANCELED = 취소
```

---

## 추천 요청 유형

```text
NEW = 신규
CHANGE = 변경
RENEWAL = 재계약
RERECOMMENDATION = 재추천
```

---

## 추천 요청 등록 필드

필수:

- 요양원
- 요청 유형
- 요청일
- 요청 사유

선택:

- 희망 전문과목
- 희망 방문 요일
- 기존 촉탁의 유무
- 특정 요청사항
- 첨부파일
- 내부 메모

---

## 목록 화면 컬럼

| 컬럼 | 설명 |
|---|---|
| 요청일 | requestedAt |
| 요양원 | facility.name |
| 요청 유형 | requestType |
| 희망 전문과목 | requestedSpecialty |
| 상태 | status |
| 담당자 | createdBy |
| 최근 수정일 | updatedAt |

---

## 상세 화면 구성

1. 요청 기본정보
2. 요양원 정보 요약
3. 요청 사유
4. 희망 조건
5. 상태 변경 영역
6. 첨부파일
7. 내부 메모
8. 상태 변경 이력

---

## 상태 변경 규칙

기본 흐름:

```text
접수 → 검토중 → 후보추천완료 → 요양원선택대기 → 지정완료 → 계약등록완료 → 종결
```

허용:

- 언제든 취소 가능
- 종결된 건은 SUPER_ADMIN만 재오픈 가능
- 계약등록완료 전에는 종결 금지
- 후보가 없는 상태에서 후보추천완료로 변경 금지

---

## 구현 지시

1. 추천 요청 CRUD API를 구현한다.
2. 추천 요청 목록 페이지를 만든다.
3. 추천 요청 등록 페이지를 만든다.
4. 추천 요청 상세 페이지를 만든다.
5. 상태 변경 액션을 구현한다.
6. 상태 변경 시 AuditLog를 남긴다.
7. 추천 요청과 요양원 관계를 UI에 표시한다.
8. 추천 요청 등록 시 요양원 검색/선택 기능을 제공한다.

---

## 예외 처리

- 존재하지 않는 요양원 선택
- 필수 요청 사유 누락
- 잘못된 상태 변경
- 권한 없는 상태 변경
- 종결 건 수정 시도
